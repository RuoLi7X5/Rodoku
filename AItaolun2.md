# AItaolun2：Rank Logic（秩逻辑）与 Unique Rectangle（UR）重构设计摘要

> 目标：不把数学公式“硬编码”进求解器，而是通过**输入表示 + 架构选择 + 训练教材（数据与损失）**，让模型学会识别秩逻辑与致命结构（多解陷阱）。

---

## 0. 核心概念与对应建模

- **秩逻辑（Rank Logic）**：核心公式 \(R = L - T\)。在图论视角上，接近**覆盖/闭环**结构（类似 Set Cover / 局部稠密子图 + 约束覆盖）。
- **致命结构（Unique Rectangle / UR）**：依赖“数独必须唯一解”的元规则，属于**反向逻辑**：某些填法会导致多解，因此必须避免/破坏。

---

## 1. Rank Logic：数据结构从“棋盘”升级为“异构图”

### 1.1 为什么不能只用 \(9\times9\) 图像
传统棋盘图像/单通道数字表示，难以表达：
- **Truth（强区域）** vs **Link（弱区域）** 的覆盖关系
- 候选数层面的“最小运算单元”
- 跨行/列/宫的长距离链式关联

### 1.2 图输入：Heterogeneous Graph（异构图）
用 GNN/Transformer 接受更结构化的输入，至少包含三类节点：

- **Cell Nodes（81）**：每个格子一个节点（`r1c1` … `r9c9`）。
- **Candidate Nodes（729）**：每个“格子-数字”的候选节点（`r2c3(5)` 等）。这是秩逻辑运算的最小单元。
- **Set Nodes（约束集合节点，建议按“区域×数字”粒度建模）**：
  - 行约束：`Row r contains digit d`（如 5R2）
  - 列约束：`Col c contains digit d`
  - 宫约束：`Box b contains digit d`
  - 这类 Set Node 本质上表示“某个区域对某个数字的覆盖集合”

> 备注：具体数量随你是否把“区域本身”和“区域×数字”拆开而变化；工程上优先保证“区域×数字”这类约束集合可被显式表达。

### 1.3 边（Edges）：表达“包含/覆盖”
主要边类型建议：
- **Candidate → Set（membership）**：若 `r2c3` 允许填 `5`，则连接 `Candidate: r2c3(5)` 与 `Set: Row2(5)` / `Col3(5)` / `Box?(5)`
- **Candidate ↔ Cell（belongs-to）**：候选属于哪个格子（可选，但有利于聚合）

边权初始化：
- 先全部置为 **1**，后续可学习边权/注意力权重。

## 2. 用注意力/消息传递模拟 \(R = L - T\) 的“秩计算”

### 2.1 直觉
秩逻辑的有效结构往往表现为：
- 某些 **Set Nodes** 只连接到少量 **Candidate Nodes**
- 图上形成明显的局部稠密子结构/闭环，使得 **Link 覆盖 Truth**

### 2.2 GNN 机制（Message Passing + Attention）
训练目标不要求网络“算公式”，而是让它学会：
- 在图上通过消息传递聚合约束信息
- 用注意力机制聚焦到“稠密子结构/闭环”

### 2.3 Rank 辅助损失（可选但推荐）
可以加入一个辅助任务，奖励模型识别：
- **节点数 L** 与 **覆盖/真值 T** 极度接近的子结构
- 输出形式可选：
  - 链关系（强链/弱链）分类
  - “候选被覆盖程度”的打分
  - 关键 Set/Candidate 的重要性（attention 监督/对齐）

---

## 3. 显式特征工程（加速学习，避免 GNN 自学太慢）

把“半成品秩特征”作为额外输入通道或图节点特征：

### 3.1 ALS（Almost Locked Sets）热力图 / 特征
- 预处理扫描全盘：找到 **N 个格子有 N+1 个候选数** 的区域
- 输出：
  - \(9\times9\) 位置热力图（哪些格子属于 ALS）
  - 或图上对相关 Cell/Candidate 节点打标

### 3.2 共轭对（Conjugate Pairs）/ 强链特征
- 标记所有“某区域某数字只剩两个位置”的强链关系
- 这是 Rank 0 的基础组件

> 思路：明确告诉模型“这里有 ALS / 强链”，让它更容易组合出 \(R=L-T\) 的结构推理。

## 4. UR（Unique Rectangle）：把“多解”当作负反馈来教模型

### 4.1 负样本训练（Negative Sampling via Uniqueness）
核心：普通监督只教“正确动作”，UR 需要教“会导致多解的坏结构”。

- **构造坏数据**：
  - 生成/采样含 UR 结构的题面
  - 甚至可人为制造“双解数独”作为训练样本
- **标注策略**：
  - 在“破坏 UR 的关键点”上：标记为必须填（例如 Probability=1.0）
  - 在“构成 UR 陷阱的动作”上：标为极错（Probability=0.0，或更大负 loss）
- **效果**：模型学会“看到 2×2 四角由两个数字组成的矩形模式时，必须破坏，否则被惩罚”

### 4.2 全局唯一性感知器（Global Uniqueness Sensor）
架构上加辅助分支，让模型具备“是否滑向多解”的全局感知：
- **输入**：当前盘面（建议用候选张量/图表示）
- **输出**：一个标量（例如：1=唯一解，>1=多解；或二分类：唯一/非唯一）
- **训练目标**：当策略网络尝试走入 UR 陷阱时，该分支应给出强烈“非唯一”信号，从而抑制该动作

## 5. 推荐的整体执行路线图（从输入到训练）

### 5.1 输入层升级（关键）
放弃纯 Image Input，至少使用：
- **Boolean Tensor \(9\times9\times9\)**：直接表示每格每数候选是否存在  
原因：秩逻辑基于候选数；若只输入已知数，模型还要先“自推候选”，浪费容量。

可选进一步升级：
- 将候选张量转为上文的**异构图**输入（Cell/Candidate/Set）

### 5.2 架构选择
优先：
- **Transformer** 或 **GNN**（天然全局视野/注意力）
原因：秩逻辑和 UR 都经常跨越行/列/宫的长距离关系；CNN 需要非常深才有足够感受野。

### 5.3 混合损失（Hybrid Loss）
建议形态：
\[
Loss = L_{policy} + \lambda_1 L_{rank} + \lambda_2 L_{UR}
\]

- **\(L_{policy}\)**：策略监督（预测下一步人类高手/求解器会下哪一步）
- **\(L_{rank}\)**：辅助任务（强/弱链、覆盖结构、关键节点重要性等）
- **\(L_{UR}\)**：若一步导致终局变成多解/触发致命结构，则给予明显惩罚（自对弈或反事实评估）

---

## 6. 落地建议（“环境和教材”优先于“硬写公式”）

- 不要强行把 \(R=L-T\) 写成固定规则塞进网络；优先让网络在结构化输入上通过注意力学习到“覆盖/闭环”模式。
- UR 的关键是把“多解”作为明确的负反馈：要有能稳定产生 UR 负样本的管线，以及能评估唯一性的辅助信号（或外部 oracle）。
